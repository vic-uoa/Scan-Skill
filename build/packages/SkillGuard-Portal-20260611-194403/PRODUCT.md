# Product

## Register

product

## Users

SkillGuard serves bank testing teams, security reviewers, AI automation owners, and skill maintainers who need to evaluate local Agent Skills before team sharing, installation, CI usage, or internal approval. They work in controlled desktop or intranet environments and need fast confidence without exposing sensitive paths, tokens, reports, or production system details.

## Product Purpose

SkillGuard is a local security admission scanner for Agent Skills. It identifies static risk evidence, applies an internal security knowledge base, supports user-facing and maintainer-facing reports, and can optionally use an OpenAI-compatible model to generate clearer remediation advice. Success means a reviewer can upload or select a skill folder, confirm whether `SKILL.md` exists, choose a scan path, produce an auditable report, and understand the next action.

## Brand Personality

Precise, calm, professional. The product should feel like an internal security control surface: technically capable, restrained, readable, and trustworthy under review pressure.

## Anti-references

Avoid marketing-heavy SaaS pages, decorative dashboards, vague AI-assistant styling, dark terminal cosplay, oversized hero statistics, and interfaces that hide the actual scan workflow behind explanatory content.

## Design Principles

Show the workflow first.
Make security evidence inspectable before asking users to scan.
Keep AI as an explicit assisted mode, never as a hidden decision-maker.
Prefer restrained density over decorative complexity.
Expose audit-ready outputs and command mapping clearly.

## Accessibility & Inclusion

Target WCAG AA contrast for body text and controls. Support keyboard-operable dialogs and controls. Provide reduced-motion alternatives for carousel and progress motion. Use status text and deterministic labels so security reviewers can understand the flow without relying on color alone.
