# 行内规范规则域映射

| 规则域 | 规则前缀 | 覆盖内容 |
|:---|:---|:---|
| 通用安全 | `NORM-GEN` | 调试入口、Mock 开关、后门、逻辑炸弹、可信组件 |
| Agent/LLM 安全 | `AGENT` | 提示注入、远程指令依赖、不可信工具输出、过度代理权限、模型/工具输出直接执行 |
| 认证安全 | `NORM-AUTH` | 弱口令、默认密码、认证过程、MFA、证书 |
| 会话安全 | `NORM-SESSION` | Cookie Secure/HttpOnly、会话超时、注销失效 |
| 权限控制 | `NORM-ACCESS` | 水平越权、垂直越权、服务端鉴权、流程绕过 |
| 输入输出安全 | `NORM-INPUT` | 类型、长度、范围、危险字符、输出编码 |
| 数据安全 | `NORM-DATA` | 脱敏、传输加密、存储加密、弱算法、数据采集 |
| 文件安全 | `NORM-FILE` | 上传、下载、路径遍历、目录浏览、外部引用 |
| API 安全 | `NORM-API` | 认证授权、限频、幂等、最小响应、审计日志 |
| 客户端安全 | `NORM-CLIENT` | Root/越狱、模拟器、完整性、混淆、密钥保护 |
| 交易安全 | `NORM-TRADE` | 防重放、唯一交易号、金额币种校验、二次认证 |
| 日志报告安全 | `NORM-LOG` | 日志防泄露、防篡改、报告与缺陷流转脱敏 |

规则结果分为：

- `confirmed`：证据明确，默认整改。
- `probable`：高度疑似，需要负责人确认。
- `needs_review`：上下文不足，需要人工复核。

## 规则使用约束

- `NORM-*` 规则用于识别“当前实现可能未满足行内规范”的场景，不应用来处罚安全需求库、规则库或修复建议本身。
- 当命中文本来自 `references/`、`security-analysis/`、`examples/`、`payloads/` 时，必须结合文件角色和上下文判断，不能只按关键词输出高危。
- 安全需求描述、测试 payload、修复示例、字段类型声明、报告摘要字段属于高频误报来源，详见 `false-positive-taxonomy.md`。
- 报告字段中的真实接口、用户号、认证上下文、请求参数属于“报告泄露”专项，不应混同为运行时代码漏洞。
- 对 Agent/LLM 类规则，只有当外部内容、工具输出或模型输出被当作指令/代码执行，或高风险动作缺少人工确认时，才进入最终报告；单纯描述提示注入风险或安全要求不应算漏洞。
